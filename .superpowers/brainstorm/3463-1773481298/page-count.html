How many pages in the booking flow?

AI generates the plan — so how few screens can we get away with?

A

3 Pages: Input → Plan → Book

Page 1 — "Tell us your cruise": Ports + dock times + travelers + budget + vibe preferences. One form, one page.

Page 2 — "Your Shore Plan": AI-generated plan with port tabs, activity cards, swap buttons. This IS the schedule — no separate Activities or Schedule step. User refines until happy.

Page 3 — "Book It": Cost summary + contact info + submit quote.

Minimum friction. Each page has a clear purpose. The AI does the heavy lifting on page 2.

B

2 Pages: Smart Form → Plan+Book

Page 1 — "Your Cruise": Same input form (ports, travelers, budget, vibe).

Page 2 — "Your Shore Plan": AI plan with port tabs AND the booking form at the bottom. Review + book on the same page. No third click.

Ultra-minimal. Risk: page 2 might feel long/overwhelming with plan + form together.

C

Single Page: Everything on one scroll

One long page with sections that reveal progressively:

  1. Enter cruise details (top)
  2. Click "Generate Plan" → AI section appears below
  3. Refine activities → cost updates live
  4. Contact form appears at bottom → submit

No page transitions at all. Risk: very long page, hard to track progress.
